Good p-Tuples

Let \(p\) be a fixed odd prime. A \(\textit{p}\)-tuple \(\left(a_1, a_2, a_3, \dots, a_p\right) \) of integers is said to be \(\textit{good}\) if (i) \(0\leq a_i \leq p-1\) for all \(i\), and (ii) \(a_1+a_2+a_3+\dots+a_p\) is not divisible by \(p\), and (iii) \(a_1a_2 + a_2a_3 + a_3a_4 + \dots + a_pa_1\) is divisible by \(p\). Determine the number of good \(\textit{p}\)-tuples.

Solution

The answer is \(p^{p-2}\left(p-1\right)\).

Call a \(\textit{p}\)-tuple \(\textit{close}\) if it satisfies at least the first two conditions but not necessarily the third. Furthermore, define a \(\textit{cyclic shift}\) of a \(\textit{p}\)-tuple \(\left(a_1, a_2, a_3, \dots, a_p\right)\) as the tuple obtained from adding a constant \(k \left(0 \leq k \leq p - 1\right)\) to each element and taking the values modulo \(p\).

In particular, for any given close \(\textit{p}\)-tuple, there exists \(p\) cyclic shifts. We then claim that exactly one of these cyclic shifts is good. If we let \(A_0 = \left(a_1, a_2, a_3, \dots, a_p\right)\) and \(A_k = \left(a_1 + k, a_2 + k, a_3 + k, \dots, a_p + k\right)\), then note that \[ \sum_{i=1} ^p\left(a_i + k\right) = pk + \sum_{i=1}^p a_i \equiv \sum_{i=1}^p a_i \not\equiv 0 \pmod{p}, \] and \[\begin{align*} \sum_{i=1}^p\left(a_i + k\right)\left(a_{i+1}+k\right) &=\sum_{i=1}^p\left(a_ia_{i+1} + ka_i + ka_{i+1} + k^2\right) \\ &= pk^2 + 2k\sum_{i=1}^p a_i + \sum_{i=1}^p a_ia_{i+1} \\ &\equiv 2k\sum_{i=1}^p a_i + \sum_{i=1}^p a_ia_{i+1}, \end{align*}\] where indices are taken cyclically in the latter statement. Since \(p\nmid \sum_{i=1}^p a_i\) and \(p\) is an odd prime, there must exist a unique value of \(k\) such that \[2k\sum_{i=1}^p a_i + \sum_{i=1}^p a_ia_{i+1} \equiv 0 \pmod{p}.\] This proves the claim.

Hence, the total number of good \(\textit{p}\)-tuples is \(\frac{1}{p}\)-th of the number of close \(\textit{p}\)-tuples. Constructive counting gives us a count of \(p^{p-1} \left(p-1\right)\) close \(\textit{p}\)-tuples, so the number of good \(p\)-tuples is \(p^{p-2} \left(p-1\right)\).

\(\blacksquare\)

Motivation

The motivation for the solution is to try to reduce the complexity of the solution by analyzing a small subset of the problem. Clearly, the third condition is the hardest to deal with, so we try to only focus on the first two conditions, leading to the idea of a "close" \(\textit{p}\)-tuple.

From there, experimentation should lead to the key observation that only one cyclic shift for any close \(\textit{p}\)-tuple satisfies the third condition.
