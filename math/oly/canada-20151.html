Functional Inequality in N

Let \(\mathbb{N} = \{1, 2, 3, \dots\}\) be the set of positive integers. Find all functions \(f\), defined on \(\mathbb{N}\) and taking values in \(\mathbb{N} \), such that \((n-1)^2 < f(n)f(f(n)) < n^2 + n\) for every positive integer \(n\).

Solution

We claim that the only solution to \(f\) is \(f(n)=n\). Clearly, the inequality \[\ (n-1)^2 < f(n)f(f(n)) = nf(n) = n^2 < n^2 + n \] holds. We then show that \(f(n) = n\) for each \(n \in \mathbb{N}\).

To begin, we prove by induction that \(f(n)\geq n\) for each \(n \in \mathbb{N}\). The base case is \(n = 1\), where in which \(0 < f(1)f(f(1)) < 2\). This implies that \(f(1) = 1\). Now suppose that \(f(k)\geq k\) for each \(1 \leq k \leq n - 1\), and assume FTSOC that \(f(n) < n\). Since \(f(n)\) is in \(\mathbb{N}\), we know that \[\begin{align*} &\quad f(n) < n \\ \iff&\quad f(n) \leq n - 1 \\ \implies&\quad f(n)^2 \leq (n-1)^2 < f(n)f(f(n)) \\ \implies&\quad f(n) < f(f(n)). \end{align*}\]

However note that because \(1 \leq f(n) \leq n-1\), by the hypothesis, \(f(f(n)) \geq f(n)\). This is a contradiction, so \(f(n) \geq n\) completes the induction.

We then show that \(f(n) \not> n\) for each \(n \in \mathbb{N}\). Suppose that \(f(n) > n\). We then have \[\begin{align*} &\quad f(n) > n \\ \iff&\quad f(n) \geq n + 1 \\ \implies&\quad f(n)^2 \geq (n+1)^2 > n^2+n > f(n)f(f(n)) \\ \implies&\quad f(n) > f(f(n)). \end{align*}\]

But clearly, by the fact that \(f(n) \geq n\) for each \(n\), it must be that \(f(f(n)) \geq f(n)\). This is a contradiction, so \(f(n) \not> n\).

In conclusion, we have shown that \(f(n) = n\) for each \(n\), so the only function \(f:\mathbb{N} \rightarrow \mathbb{N}\) must be \(f(n)=n\).

\(\blacksquare\)

Motivation

This is a typical garden-variety functional \(\sout{\text{equation}}\) inequality.

Observing that \(f\) is defined in \(\mathbb{N}\), we are motivated to test small values of \(n\). At first, the experimentation will make the problem smell like number theory, but after a while, we observe a recursive structure in the bounds. From here, we guess that the answer is just \(n\), and the usage of strong induction is highly motivated.

The only point of caution is to not get \(\textit{for each}\) mixed up with \(\textit{for some}\). This is the infamous point-wise trap. When we say that \(f(n) \geq n\), we must specify whether this is true for some particular value of \(n\), or for each \(n \in \mathbb{N}\). The distinction seems subtle, but the consequences are dire. For example, if \(f(n) \geq n\) is for true for each \(n\), then we can safely conclude that \(f(f(n)) \geq f(n)\), whereas if the argument is only true for some \(n\), we cannot make such conclusions.

In fact, there are lines of logic that are even more seemingly subtle than what is described above. The famous one is the distinction between \(\textit{for each} \) and \(\textit{for all}\). Intuitively, they seem interchangeable (\(\forall\)), but they can mean completely different things depending on the context. In these cases, leaning towards for each is a good bet.
