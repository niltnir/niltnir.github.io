Color Switcher

A lock has \(16\) keys arranged in a \(4 \times 4\) array, each key oriented either horizontally or vertically. In order to open it, all the keys must be vertically oriented. When a key is switched to another position, all the other keys in the same row and column automatically switch their positions too. Show that no matter what the starting positions are, it is always possible to open this lock. (Only one key can be switched at a time.)

Solution

It suffices to show that there exists a sequence of moves such that any of the keys can be switched individually without altering the rest of the configuration.

Select any key to individually switch and call this the \(\textit{main}\) key. Now consider all the keys that occupy the same row or the same column as this key. Together, there are \(7\) such keys including the main key. We claim that our sequence of moves consists of switching each of these keys once regardless of order.

To prove that this works, we count the number of times each key gets switched. There are three cases to consider: the main key, the \(6\) other keys to switch, and the rest of the keys left untouched.

For the main key, the position changes each time we switch one of the \(7\) keys. Since \(7\) is odd, by the end of the operation, it will switch to a different position.

Meanwhile, for the other \(6\) keys, a key either shares the same row or the same column as the main key. WLOG, if the key shares the same row with the main key, then when the \(7\) keys get switched, only the keys in the same row affect our key. Hence, any such key gets switched \(4\) times, returning them to their original positions.

Lastly, for the untouched keys, a key will share the same row with one of the \(6\) keys, and share the same column with another of the \(6\) keys. This means that untouched keys are switched a total of \(2\) times, so they too will switch back to their original positions.

We have shown that only the main key gets switched to a different position after the sequence of moves, so we are done.

\(\blacksquare\)

Commentary

The solution can easily be generalized for any even by even array. On the other hand, when the array has odd dimensions, there seem to be certain invariants preventing potential solutions. I haven't investigated the problem enough, so I can't say for sure.
